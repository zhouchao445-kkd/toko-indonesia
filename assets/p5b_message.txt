## 1. 后端 API 扩展(写到 backend/src/routes/)

### a. admin-orders.ts(扩展,P4-A 已有 review-proof + ship,补全列表/详情/取消)
- GET /api/admin/orders — 列表(支持筛选:状态/时间范围/会员ID/订单号搜索/分页)
- GET /api/admin/orders/:id — 详情(含会员/地址/商品/凭证/物流完整信息)
- POST /api/admin/orders/:id/cancel — 超管取消(approved 前可取消,取消时恢复库存)
- 已有 review-proof 和 ship 保留
- 全部走 logOperation
- 权限:order_manager / super_admin 可写(取消/审核/发货),customer_service 只读,finance_manager 只读(看财务相关)

### b. admin-payment-proofs.ts(新,凭证审核独立列表)
- GET /api/admin/payment-proofs — 待审列表(status=pending)+ 全部列表(status 筛选)
- GET /api/admin/payment-proofs/:id — 凭证详情(含图片 URL)
- POST /api/admin/payment-proofs/:id/approve — 同意(订单 → approved)
- POST /api/admin/payment-proofs/:id/reject — 拒绝(订单 → rejected + 拒绝原因)
- 全部走 logOperation
- 权限:order_manager / super_admin

### c. admin-reviews.ts(新,评价审核)
- GET /api/admin/reviews — 列表(全部 + 状态筛选:pending/visible/hidden)
- GET /api/admin/reviews/:id — 评价详情
- PATCH /api/admin/reviews/:id/status — 改状态(visible/hidden,**没有删除**)
- 全部走 logOperation
- **先审后公**规则:商品详情页只显示 status='visible' 的评价(P3-B 商品详情页要对应改查询)
- 权限:product_manager / super_admin 可写,全部角色可读

### d. admin-members.ts(新,会员管理)
- GET /api/admin/members — 列表(支持筛选:注册时间/状态 enabled/disabled/余额范围/分页)
- GET /api/admin/members/:id — 详情(基本信息 + 地址列表 + 订单历史 + 余额流水)
- PATCH /api/admin/members/:id/status — 启用/禁用(disabled 后无法登录)
- 全部走 logOperation
- 权限:customer_service / super_admin 可写(只禁用/启用,不删)

### e. admin-balance.ts(新,余额管理 — 关键)
- 余额变更类型分两种:
  1. **直接修改**(super_admin / finance_super_admin 专属,不走审核) — 调账、退款、补偿
  2. **审核流程**(order_manager 申请,finance_super_admin / super_admin 审核) — 充值、其他
- 建议表设计:`balance_change_requests` 表(id / member_id / amount / type / reason / status / requester_id / reviewer_id / created_at / reviewed_at)
  - type: 'direct' (直改) | 'review' (审核)
  - status: 'pending' | 'approved' | 'rejected'
- API:
  - GET /api/admin/balance-requests — 待审列表(只 status=pending)
  - GET /api/admin/balance-requests/:id — 详情
  - POST /api/admin/balance-requests — 申请变更(member_id / amount / reason)
  - POST /api/admin/balance-requests/:id/approve — 审核通过(直改余额 + 写余额流水)
  - POST /api/admin/balance-requests/:id/reject — 审核拒绝
  - POST /api/admin/members/:id/balance/direct — **直改余额**(只 super_admin / finance_super_admin,不走审核)
- 全部走 logOperation
- 权限矩阵:
  - direct 直改: super_admin / finance_super_admin
  - 申请变更: order_manager
  - 审核: super_admin / finance_super_admin
  - 查看: 全部后台角色
