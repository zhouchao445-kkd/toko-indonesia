- POST /api/admin/conversations/:id/assign — 分配客服 { customer_service_id }

- 全部走 logOperation

#### c2. backend/src/routes/admin-messages.ts(新,客服消息 + 轮询 fallback)
- GET /api/admin/conversations/:id/messages — 拉取消息(支持 since=<timestamp> 增量拉取)
- POST /api/admin/conversations/:id/messages — 发送消息 { content, type(text/image) }
- 全部走 logOperation
- 权限:customer_service / super_admin

#### c3. backend/src/lib/chat-polling.ts(新,前端 5s 轮询工具)
- export function useChatPolling(conversationId, onMessage)
- 5s 轮询拉取新消息(用 since=<last_timestamp>)
- 30s 心跳(记录最后在线时间)
- 自动重连:网络断开 3 次后 → 退避到 10s,恢复后回 5s
- 不是 WebSocket 客户端,只是基于 fetch 的轮询 hook

#### c4. backend/src/lib/heartbeat.ts(新,前后端心跳)
- 后端: GET /api/admin/heartbeat — 返回 { server_time, status: 'ok' }
- 前端 30s 调一次,用于检测后端是否在线
- 失败 3 次 → UI 提示"连接中断,正在重试"

## 8. 关键提示

- 客服模块**核心是"消息入库" + "轮询"**,不是 WebSocket。这跟 P2 时代定的"风险 4 方案 A"一致。

- 5s 轮询频率要可配置(可改成 10s / 30s),默认 5s
- 心跳 30s 失败 3 次 → 提示"连接中断"
- 三语翻译覆盖所有按钮 / 状态 / 弹窗
- 客服消息气泡:左 = 会员(灰)/ 右 = 客服(蓝),时间显示
