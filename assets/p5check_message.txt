## A. 安全类(高优先级,先查)

### A1. 权限越权 — 后台写操作是否真的被 requirePermission 拦截?
- 列出所有后台写路由(POST/PUT/DELETE/PATCH)
- 对每个路由 grep requirePermission,确认 100% 覆盖
- 对每个写操作,检查是否做了角色检查(不是只检查登录)
- 重点查:settings 修改、PII 数据访问(用户手机/邮箱/地址)、发货操作

### A2. 会员侧越权 — 会员能否访问别人的订单/地址/购物车?
- 查 orders.ts / users.ts / cart.ts / payment-proofs.ts
- 对每个 user_id 过滤的位置,确认不是只查主键而没查 user_id
- 重点查:GET /api/orders/:id(必须 .where user_id = current)、GET /api/users/addresses(必须 .where user_id = current)

### A3. SQL 注入 / 原始查询
- grep $queryRaw / $executeRaw / Prisma.raw
- 确认没有字符串拼接 user input 进 SQL

### A4. 文件上传安全
- multer 配置:文件大小限制、文件类型白名单(只 jpg/png/webp)、文件名防路径穿越
- /uploads/* 静态托管是否暴露了 .git / .env / 其他敏感文件?

### A5. 密码 / JWT 安全
- 密码是否 bcrypt 哈希(不是明文 / SHA1)
- JWT secret 长度 ≥ 32 字符,且从 .env 读(没硬编码)
- JWT 过期时间合理(后台 8h?会员 24h?refresh token?)

### A6. 凭证上传 — 越权上传别人的订单凭证?
- POST /api/orders/:id/payment-proofs 必须验证订单归属

## B. 数据一致性(中优先级)

### B1. 软删一致性
